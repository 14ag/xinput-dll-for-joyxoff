TASK GOAL

Create a modified XInput proxy DLL (x86 only for now) based on x360ce’s open-source wrapper.

The proxy must:

Forward real XInput calls to the system XInput DLL.

Inject improved device enumeration using RawInput + HID + SetupAPI (SetupDi).

Treat new or remapped devices as stable even when device instance paths change.

Fix JoyXoff’s detection weakness by making XInputGetState more resilient.

Do not use x360ce’s GUI or its DirectInput-based mapping engine. Only reuse the wrapper DLL code structure.

GENERAL ARCHITECTURE

Build a custom xinput1_3.dll that exports the standard XInput API.

Internally load the real system xinput1_3.dll (from System32/SysWOW64).

Implement a “Device Resolver Layer” that performs:

RawInput device enumeration

HID capabilities query

Device path hashing

Stable logical controller indexing

When a game calls XInputGetState, the proxy:

maps the requested dwUserIndex to the correct physical controller

calls the real XInput function

returns the real XInput packet

falls back to RawInput-based synthesis if real XInput fails to detect the device
